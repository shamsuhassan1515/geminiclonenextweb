# FlyEnv 启动项目配置说明

## 项目概述

本项目是一个 ChatGPT Web 应用，包含**前端**和**后端**两个独立服务，需要同时运行才能正常使用。

| 服务 | 目录 | 端口 | 技术栈 |
|------|------|------|--------|
| 前端 | `/` | 1002 | Vue 3 + Vite |
| 后端 | `/service` | 3002 | Express + TypeScript |

---

## FlyEnv 配置步骤

### 第一步：添加前端项目

1. 打开 FlyEnv，点击「添加项目」
2. 选择文件：`/Users/aydinkaya/开发/chatgpt-web-midjourney-proxy/package.json`
3. 配置如下：

| 配置项 | 值 |
|--------|-----|
| Scripts | `dev: vite` |
| TCP 端口 | `1002` |
| 项目名称 | ChatGPT Web 前端 |

4. 点击保存

### 第二步：添加后端项目

1. 再次点击「添加项目」
2. 选择文件：`/Users/aydinkaya/开发/chatgpt-web-midjourney-proxy/service/package.json`
3. 配置如下：

| 配置项 | 值 |
|--------|-----|
| Scripts | `dev` |
| TCP 端口 | `3002` |
| 项目名称 | ChatGPT Web 后端 |

4. 点击保存

### 第三步：启动顺序（重要！）

**必须按以下顺序启动：**

1. **先启动后端**（端口 3002）→ 等待显示 `Server is running on port 3002`
2. **再启动前端**（端口 1002）→ 等待显示 `Local: http://localhost:1002/`

### 第四步：访问应用

浏览器打开：http://localhost:1002/

## 配置要点

### 1. 环境变量配置

后端服务需要正确配置 `.env` 文件，位于 `/service/.env`。

**必须配置的关键变量：**

```env
# OpenAI API 配置
OPENAI_API_KEY=sk-xxxxx
OPENAI_API_BASE_URL=https://api.openai.com

# 以下 SERVER 变量必须有值，否则后端启动失败
LUMA_SERVER=https://api.openai.com
SUNO_SERVER=https://api.openai.com
RUNWAY_SERVER=https://api.openai.com
RUNWAYML_SERVER=https://api.openai.com
KLING_SERVER=https://api.openai.com
VIGGLE_SERVER=https://api.openai.com
IDEO_SERVER=https://api.openai.com
PIKA_SERVER=https://api.openai.com
PIXVERSE_SERVER=https://api.openai.com
UDIO_SERVER=https://api.openai.com
MJ_SERVER=https://api.openai.com
```

### 2. 依赖安装

确保两个目录都已安装依赖：

```bash
# 前端依赖
cd /Users/aydinkaya/开发/chatgpt-web-midjourney-proxy
pnpm install

# 后端依赖
cd /Users/aydinkaya/开发/chatgpt-web-midjourney-proxy/service
pnpm install
```

### 3. Node.js 版本要求

- Node.js >= 18.0.0
- 推荐使用 pnpm 作为包管理器

## 常见问题与解决方案

### 问题 1：FlyEnv 添加项目无反应

**原因**：可能存在多个 FlyEnv 配置文件冲突

**解决方案**：
- 确保项目根目录没有多余的配置文件（flyenv.yaml、.flyenv 等）
- 只通过 FlyEnv 界面添加项目，不要手动创建配置文件

### 问题 2：后端启动失败 - "Host should not be empty"

**原因**：`.env` 文件中的 SERVER 变量为空字符串，`express-http-proxy` 无法处理

**解决方案**：
- 确保 `/service/.env` 中所有 `*_SERVER` 变量都有有效值
- 空字符串 `""` 和未设置是不同的，空字符串也会导致错误

### 问题 3：前端页面一直转圈加载

**原因**：
1. 后端未启动
2. 路由配置冲突

**解决方案**：
1. 确保后端服务已启动（端口 3002）
2. 检查浏览器控制台是否有路由错误

### 问题 4：路由错误 - "A route named 'xxx' has been added as a child of a route with the same name"

**原因**：Vue Router 中父路由和子路由使用了相同的名称

**解决方案**：
检查 `/src/router/index.ts`，确保父路由名称与子路由名称不同：

```typescript
// 错误示例
{
  path: '/music',
  name: 'music',  // 父路由名称
  children: [
    {
      path: '/music/:uuid?',
      name: 'music',  // 子路由名称相同，报错！
    }
  ]
}

// 正确示例
{
  path: '/music',
  name: 'MusicRoot',  // 父路由名称改为不同
  children: [
    {
      path: '/music/:uuid?',
      name: 'music',  // 子路由名称
    }
  ]
}
```

### 问题 5：API 请求失败

**原因**：
1. 后端未启动
2. API Key 未配置或无效

**解决方案**：
1. 确保后端服务运行中
2. 在 `.env` 文件中配置有效的 `OPENAI_API_KEY`

## 手动启动方式

如果 FlyEnv 无法正常工作，可以使用终端手动启动：

```bash
# 终端 1 - 启动后端
cd /Users/aydinkaya/开发/chatgpt-web-midjourney-proxy/service
pnpm run dev

# 终端 2 - 启动前端
cd /Users/aydinkaya/开发/chatgpt-web-midjourney-proxy
pnpm run dev
```
